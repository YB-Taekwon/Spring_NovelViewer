# 🗒 Novel Viewer  
웹소설 콘텐츠 플랫폼  

<br>  

## 주요 기능  

### ✅ 인증 관리 (Auth)
- [ ] **회원 가입**
  - 아이디, 비밀번호, 이메일, 이름을 입력하여 가입할 수 있습니다.
  - 아이디는 중복될 수 없으며(`UNIQUE`), 가입 시 기본 권한은 `USER`입니다.
  - JWT를 사용해 인증을 처리합니다.
  - Redis와 Mailgun을 통해 이메일 인증을 완료해야 회원 가입이 완료됩니다.
- [x] **로그인**
  - 가입 시 입력한 아이디와 비밀번호로 로그인할 수 있습니다.
- [x] **로그아웃**
  - Redis를 통해 토큰을 블랙리스트에 저장하여 인증을 차단합니다.
- [ ] **소셜 로그인**
  - OAuth2를 기반으로 소셜 계정을 통한 간편 로그인을 지원합니다.

<br>  

### ✅ 콘텐츠 관리 (Novel)
- [x] **소설 조회**  
  - 로그인한 사용자는 모든 작품을 조회할 수 있습니다.  
  - 작가명, 작품명으로 검색이 가능합니다.
  - 카테고리 필터와 페이징 기능을 지원합니다.
- [x] **소설 등록**
  - 작가 권한(`AUTHOR`)을 가진 사용자만 소설을 등록할 수 있습니다.
  - 제목, 소설 소개, 카테고리를 입력하고, 표지 이미지를 업로드할 수 있습니다.
  - 작품마다 `content_id`라는 UUID 기반의 외부 노출용 고유 식별자가 자동 부여됩니다.
- [x] **소설 수정**  
  - 작가는 자신이 등록한 소설과 표지 이미지를 수정할 수 있습니다.
- [x] **소설 삭제**  
  - 작가는 자신이 등록한 소설을 삭제할 수 있습니다.
  - 관리자는 모든 작품을 삭제할 수 있습니다.
- [x] **선호 작품 (북마크)**  
  - 로그인한 사용자는 원하는 소설을 북마크에 추가할 수 있습니다.
  - 동일한 작품은 한 번만 북마크할 수 있으며, 중복 등록은 불가능합니다.
  - Redis의 `SADD` 명령어를 사용하여 북마크에 추가하거나, `SREM` 명령어를 사용하여 북마크에서 제거합니다.

<br>  

### ✅ 회차 관리 (Episode)
- [x] **회차 조회**  
  - 로그인한 사용자는 소설의 회차를 열람할 수 있습니다.  
- [x] **회차 등록**
  - 작가 권한(`AUTHOR`)을 가진 사용자만 회차를 등록할 수 있으며, 본인이 등록한 소설의 회차만 등록할 수 있습니다.
  - 제목, 내용을 입력하여 회차를 등록할 수 있습니다.  
- [x] **회차 수정**  
  - 작가는 자신이 등록한 회차를 수정할 수 있습니다.
- [x] **회차 삭제**  
  - 작가는 자신이 등록한 회차를 수정하거나 삭제할 수 있습니다.
  - 관리자는 모든 회차를 삭제할 수 있습니다.

<br>  

### ✅ 댓글 관리 (Comment)
- [x] **댓글 조회**  
  - 로그인한 사용자는 회차에 댓글 및 대댓글을 열람할 수 있습니다.
- [x] **댓글 작성**  
  - 로그인한 사용자는 회차에 댓글 및 대댓글을 작성할 수 있습니다.
  - 대댓글은 1단계만 허용됩니다.
- [x] **댓글 수정**  
  - 로그인한 사용자는 자신이 작성한 댓글만 수정할 수 있습니다.   
- [x] **댓글 삭제**  
  - 사용자는 자신이 작성한 댓글만 삭제할 수 있습니다.  
  - 관리자는 모든 댓글을 삭제할 수 있습니다.  

<br>  

### ✅ 회원 관리 (User)
- [x] **회원 정보 조회**
  - 로그인한 사용자는 회원가입 시 등록한 정보를 조회할 수 있습니다.
- [x] **북마크 목록 조회**
  - 로그인한 사용자는 북마크한 소설 목록을 조회할 수 있습니다.
  - Redis의 Set 자료구조를 활용하며, 사용자의 로그인 ID를 key로 갖습니다.
- [x] **내가 단 댓글 조회**
  - 로그인한 사용자는 본인이 등록한 모든 댓글을 볼 수 있습니다.
- [x] **작가 권한 요청**
  - 로그인한 사용자는 작가 권한을 요청할 수 있으며, 관리자의 승인 하에 작가로 활동할 수 있습니다.
  - 작가명(필명)을 입력하여 작가 권한을 요청할 수 있습니다.

<br>  

### ✅ 관리자 (Admin)
- [x] **작가 권한 승인**
  - 관리자 권한(`ADMIN`)을 가진 사용자만 권한을 승인할 수 있습니다.
